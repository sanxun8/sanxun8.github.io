## 构建流程

- Vite 的构建流程分为两个主要阶段:开发服务器和生产构建。
- 开发服务器阶段:
  - Vite 使用 esbuild 作为开发服务器的底层构建引擎。
  - esbuild 是一个高性能的 JavaScript 和 CSS 打包工具,它使用 Go 语言编写,可以提供极快的构建速度。
  - 在开发阶段,Vite 会利用 esbuild 的快速模块加载和 HMR (热模块替换) 特性,为开发者提供流畅的开发体验。
- 生产构建阶段:
  - 在生产构建阶段,Vite 会使用 Rollup 作为底层的打包工具。
  - Rollup 是一个专注于 ES 模块的 JavaScript 打包器,它可以生成高度优化的产品构建包。
  - Vite 利用 Rollup 的插件系统,结合 esbuild 的速度优势,提供了一套完整的生产构建流程。

## 原理和集成

- esbuild 集成:
  - Vite 利用 esbuild 的 JS/CSS 转换和模块解析能力,提供快速的开发服务器体验。
  - esbuild 可以快速编译 ES 模块,并支持 HMR,这有助于提高开发效率。
- Rollup 集成:
  - 在生产构建阶段,Vite 会使用 Rollup 作为底层的打包工具。
  - Rollup 擅长于生成高度优化的产品构建包,并提供丰富的插件生态。
  - Vite 利用 Rollup 的插件系统,可以在构建过程中应用各种转换和优化。
- 其他集成:
  - Vite 还集成了其他工具,如 PostCSS、TypeScript 等,以提供更完整的开发体验。
  - 这些工具的集成使 Vite 能够处理各种类型的资源,并提供丰富的功能扩展。

## 性能优化

- Vite 的性能优势主要来自于 esbuild 的快速编译能力和 Rollup 的优化输出。
- 在开发阶段,esbuild 的高速模块加载和 HMR 机制,可以大大提高开发效率。
- 在生产构建阶段,Rollup 的优化输出,如 tree-shaking、code splitting 等,可以生成体积更小、性能更优的最终构建包。
- Vite 还提供了其他性能优化功能,如 lazy-loading、base URL 管理等,以满足不同应用场景的需求。
