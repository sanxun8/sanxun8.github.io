## CommonJS 模块化规范

### 原理:

- CommonJS 是 Node.js 中使用的模块化规范,采用同步加载模块的方式。
- 每个文件就是一个模块,内部通过 require() 函数加载其他模块,通过 exports 对象导出接口。

### 用法:

- 使用 require() 函数导入模块: const module = require('./module.js');
- 使用 exports 对象导出接口: exports.foo = 'bar';
- 也可以使用 module.exports 对象整体导出: module.exports = { foo: 'bar' };

## AMD (Asynchronous Module Definition) 模块化规范

### 原理:

- AMD 是一种异步加载模块的规范,适用于浏览器环境。
- 采用 define() 函数定义模块,通过 require() 函数异步加载模块。

### 用法:

- 使用 define() 定义模块: define(['dep1', 'dep2'], function(dep1, dep2) { ... });
- 使用 require() 加载模块: require(['module1', 'module2'], function(module1, module2) { ... });

## ES6 模块化规范

### 原理:

- ES6 模块化规范是 JavaScript 语言层面的模块系统,采用静态分析的方式。
- 使用 import 和 export 关键字进行模块的导入和导出。

### 用法:

- 使用 export 关键字导出接口: export const foo = 'bar';
- 使用 import 关键字导入模块: import { foo } from './module.js';
- 也可以使用 export default 导出默认接口: export default { foo: 'bar' };
- 然后使用 import module from './module.js'; 导入默认接口

这三种模块化规范各有特点,适用于不同的场景。CommonJS 适用于服务端 Node.js 环境,AMD 适用于浏览器环境,ES6 模块化规范是 JavaScript 语言层面的标准。理解它们的基本原理和用法将有助于你在实际开发中灵活使用模块化。
