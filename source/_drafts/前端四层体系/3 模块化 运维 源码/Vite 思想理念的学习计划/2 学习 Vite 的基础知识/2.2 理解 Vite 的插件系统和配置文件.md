## 插件系统

- Vite 的插件系统允许您扩展和自定义 Vite 的功能。
- 您可以创建自己的插件,或使用社区提供的插件。
- 插件可以在不同的生命周期钩子上执行自定义逻辑,如 config、transform、handleHotUpdate 等。

以下是一个简单的插件示例:

```javascript
// my-plugin.js
export default {
  name: 'my-plugin',
  transform(code, id) {
    if (id.endsWith('.js')) {
      return {
        code: `console.log('Transformed ${id}'); ${code}`,
        map: null // 提供可选的 source map
      }
    }
  }
}
```

- 您可以在 Vite 配置文件中注册此插件:

```javascript
// vite.config.js
import myPlugin from './my-plugin'

export default {
  plugins: [myPlugin()]
}
```

## 配置文件

- Vite 的配置文件通常命名为 vite.config.js 或 vite.config.ts。
- 配置文件导出一个 JavaScript 对象,其中包含各种配置选项。
- 常见的配置选项包括:
  - root: 项目根目录
  - base: 公共基础路径
  - mode: 运行模式
  - plugins: 插件列表
  - server: 开发服务器配置
  - build: 构建选项

以下是一个简单的配置文件示例:

```javascript
// vite.config.js
export default {
  root: 'src',
  base: '/',
  server: {
    port: 3000
  },
  build: {
    outDir: 'dist'
  }
}
```

## 环境变量和模式

- Vite 支持通过环境变量进行配置。
- 您可以在项目根目录创建 .env 文件,并在其中定义环境变量。
- Vite 会自动将以 VITE_ 开头的环境变量暴露给客户端代码。
- 您还可以在配置文件中根据不同的运行模式(如 development、production)应用不同的配置。

综上所述,Vite 的插件系统和配置文件提供了丰富的自定义和扩展能力。您可以根据项目需求灵活地配置 Vite,以满足各种场景下的开发和构建需求。
