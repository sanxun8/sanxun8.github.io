## 代码分割

- Vite 通过 ESM 和 dynamic imports 支持开箱即用的代码分割。
- 当您在代码中使用 import() 动态导入模块时,Vite 会自动进行代码分割,将动态导入的模块打包为单独的 chunk 文件。
- 示例:

```javascript
// 动态导入
const moduleA = await import('./moduleA.js')
```

- Vite 会在构建时自动进行代码分割优化,提高应用的初次加载性能。

## 静态资源处理

- Vite 可以处理各种静态资源,如图片、字体、CSS 文件等。
- 您可以直接在代码中导入这些资源:

```javascript
// 导入图片
import logo from './logo.png'

// 在 HTML 中使用
<img src={logo} />
```

- Vite 会自动处理这些静态资源,包括:
  - 为资源生成合适的 URL
  - 在构建时进行优化,如图片压缩
  - 支持 CSS 中的 url() 引用
- 您还可以通过配置文件中的 assetsInclude 选项,指定其他类型的静态资源。

## 公共基础路径

- 在部署应用程序时,您可能需要设置一个公共的基础路径。
- 您可以在配置文件中使用 base 选项来指定公共基础路径:

```javascript
// vite.config.js
export default {
  base: '/my-app/'
}
```

- 设置 base 后,Vite 会自动更新静态资源和 URL 引用,以适应新的基础路径。

## 环境变量

- Vite 支持在代码中使用环境变量。
- 您可以在项目根目录创建 .env 文件,并在其中定义环境变量。
- Vite 会自动将以 VITE_ 开头的环境变量暴露给客户端代码:

```javascript
console.log(import.meta.env.VITE_API_URL)
```

- 您还可以根据不同的运行模式(如 development、production)应用不同的环境变量配置。
