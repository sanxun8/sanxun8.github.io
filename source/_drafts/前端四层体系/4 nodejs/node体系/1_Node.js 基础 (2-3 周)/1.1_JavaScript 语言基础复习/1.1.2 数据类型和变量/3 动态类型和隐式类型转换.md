---
title: 动态类型和隐式类型转换
---

JavaScript 是一种动态类型语言,这意味着变量可以随时改变它们的数据类型。这与静态类型语言(如 Java 或 C++)不同,在静态类型语言中,变量的数据类型在编译时就确定了。

在 JavaScript 中,当操作不同类型的数据时,会发生隐式类型转换。这是因为 JavaScript 试图将操作数转换为一个"通用"的类型,以便执行操作。例如:

```javascript
let x = 5 + "2"; // 结果是 "52"，因为数字 5 被转换为字符串
let y = 5 - "2"; // 结果是 3，因为字符串 "2" 被转换为数字
```

JavaScript 使用以下规则进行隐式类型转换:

- 转换为布尔值:
  - 除了 0、''、null、undefined、NaN 和 false 之外,其他值都会被转换为 true。
- 转换为数字:
  - 字符串 -> 数字:空字符串 '' 转换为 0，非数字字符串转换为 NaN。
  - null 转换为 0，undefined 转换为 NaN。
  - 布尔值 true 转换为 1，false 转换为 0。
- 转换为字符串:
  - 数字 -> 字符串:直接将数字转换为对应的字符串表示。
  - null 转换为 "null"，undefined 转换为 "undefined"。
  - 布尔值 true 转换为 "true"，false 转换为 "false"。

这些隐式类型转换规则可能会导致一些意外的行为,因此在编程时需要注意。为了避免问题,最好显式地进行类型转换,如使用 Number()、String()、Boolean() 等函数。

总的来说,理解 JavaScript 的动态类型和隐式类型转换规则非常重要,这可以帮助你更好地理解和预测 JavaScript 代码的行为。
