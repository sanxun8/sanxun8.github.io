## 工作原理

Webpack 是一个静态资源打包工具,它的工作原理是:

- 将应用程序及其依赖关系视为一个整体,生成一个依赖关系图。
- 根据依赖关系图,将应用程序拆分成多个可单独部署的 chunk。
- 使用 loader 和插件处理各种类型的资源,如 JavaScript、CSS、图片等。
- 最终生成优化后的静态资源文件,供应用程序使用。

## 核心概念

- Entry:入口起点,指示 Webpack 应该使用哪个模块来作为构建其内部依赖图的开始。
- Output:输出选项,告诉 Webpack 在哪里输出它所创建的 bundles,以及如何命名这些文件。
- Loaders:用于对模块的源代码进行转换。Webpack 只能理解 JavaScript 和 JSON 文件,- loader 可以将其他类型的文件转换为 Webpack 可处理的有效模块。
- Plugins:可以在 Webpack 构建过程的各个阶段执行自定义任务。Plugins 可以执行范围更广的任务,如打包优化、资源管理和环境变量注入等。
- mode:用来指定当前的构建环境是 development、production 还是 none。
- target:指定要构建的目标运行环境,如浏览器、Node.js 等。
- source map:用于在开发环境中调试源码。

## 基本使用

初始化 Webpack 项目:

```shell
npm init -y
npm install webpack webpack-cli --save-dev
```

编写 Webpack 配置文件 webpack.config.js:

```javascript
module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  module: {
    rules: [
      // 各种 loader 配置
    ]
  },
  plugins: [
    // 各种 plugin 配置
  ]
};
```

运行 Webpack 命令进行构建:

```shell
npx webpack
```
