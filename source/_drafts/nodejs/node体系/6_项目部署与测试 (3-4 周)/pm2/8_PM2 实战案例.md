## 案例背景:

某公司开发了一个 Node.js 构建的 Web 应用程序,该应用需要 7x24 小时持续运行。在开发环境下,使用 node app.js 命令即可启动应用,但是在生产环境中,需要一种更加可靠、易管理的方式来运行应用。

## 使用 PM2 的解决方案:


### 安装 PM2:

在生产服务器上全局安装 PM2: npm install -g pm2

### 创建 PM2 配置文件:

在项目根目录下创建 ecosystem.config.js 文件,用于配置 PM2 应用:

```javascript
module.exports = {
  apps: [
    {
      name: 'my-app',
      script: 'app.js',
      instances: 2, // 启动 2 个实例
      env: {
        NODE_ENV: 'development'
      },
      env_production: {
        NODE_ENV: 'production'
      }
    }
  ]
}
```

### 启动应用:

使用 pm2 start ecosystem.config.js 命令启动应用

PM2 会自动监控应用程序的运行状态,并在发生异常时自动重启

### 常用 PM2 命令:

- pm2 status: 查看应用程序的运行状态
- pm2 logs: 查看应用程序的日志
- pm2 reload my-app: 热重载应用程序
- pm2 scale my-app +1: 动态增加 1 个实例
- pm2 stop my-app: 停止应用程序
- pm2 delete my-app: 删除应用程序

### 监控与报警:

- 使用 pm2 monit 命令查看应用程序的实时监控信息
- 配合 Keymetrics 提供的监控服务,可以设置更加丰富的监控报警规则

### 负载均衡:

- PM2 内置了负载均衡功能,可以自动在多个实例之间分配请求
- 通过 instances: 'max' 配置,可以让 PM2 根据 CPU 核数自动启动多个实例

通过以上步骤,就可以使用 PM2 将 Node.js 应用程序部署到生产环境,并提供高可用、易管理的运行环境。PM2 不仅可以确保应用程序的稳定运行,还提供了丰富的监控和报警功能,帮助开发者及时发现和解决问题。
