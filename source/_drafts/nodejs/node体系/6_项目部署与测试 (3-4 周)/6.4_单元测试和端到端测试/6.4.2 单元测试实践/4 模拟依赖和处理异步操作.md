---
title: 模拟依赖和处理异步操作
---

在编写单元测试时,我们经常需要处理依赖和异步操作。以下是一些示例:

## 模拟依赖:

```javascript
// api.js
export const fetchData = () => {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve({ data: 'Fetched data' });
    }, 2000);
  });
};

// component.js
import { fetchData } from './api';

export const Component = () => {
  const handleFetch = async () => {
    const response = await fetchData();
    console.log(response.data);
  };

  return (
    <div>
      <button onClick={handleFetch}>Fetch Data</button>
    </div>
  );
};

// component.test.js
import { Component } from './component';
import { fetchData } from './api';
import { render, fireEvent, waitFor } from '@testing-library/react';
import '@testing-library/jest-dom/extend-expect';

jest.mock('./api');

test('Component fetches data correctly', async () => {
  fetchData.mockResolvedValue({ data: 'Mocked data' });
  const { getByText } = render(<Component />);
  const button = getByText('Fetch Data');
  fireEvent.click(button);

  await waitFor(() => expect(console.log).toHaveBeenCalledWith('Mocked data'));
});
```

在上述示例中,我们使用 jest.mock('./api') 来模拟 api.js 中的 fetchData 函数。然后在测试中,我们使用 fetchData.mockResolvedValue() 来设置模拟的返回值。这样可以避免在单元测试中调用真正的 API 接口,从而提高测试的可靠性和速度。

## 处理异步操作:

```javascript
// asyncFunction.js
export const fetchData = () => {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve({ data: 'Fetched data' });
    }, 2000);
  });
};

// asyncFunction.test.js
import { fetchData } from './asyncFunction';

test('fetchData returns expected data', async () => {
  const response = await fetchData();
  expect(response.data).toBe('Fetched data');
});
```

在这个示例中,我们使用 async/await 语法来处理异步操作。通过 await 关键字,我们可以等待异步操作完成,并断言返回的数据是我们期望的结果。

如果你使用的是 Mocha 等基于回调的测试框架,你也可以使用 Promise 链或回调函数来处理异步操作:

```javascript
// asyncFunction.test.js
import { fetchData } from './asyncFunction';

it('fetchData returns expected data', (done) => {
  fetchData().then((response) => {
    expect(response.data).toBe('Fetched data');
    done();
  });
});
```

综上所述,在编写单元测试时,我们需要考虑如何处理依赖和异步操作。通过使用模拟依赖和 async/await 或 Promise 链等方式,我们可以确保测试用例的可靠性和可维护性。
