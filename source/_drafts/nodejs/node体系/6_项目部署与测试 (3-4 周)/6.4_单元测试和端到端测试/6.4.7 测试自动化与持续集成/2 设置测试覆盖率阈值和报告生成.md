---
title: 设置测试覆盖率阈值和报告生成
---

在 Node.js 项目中设置测试覆盖率阈值和生成测试覆盖率报告的步骤如下:


## 安装覆盖率工具:

常用的 Node.js 代码覆盖率工具有 Istanbul 和 NYC。这里以 NYC 为例进行说明。

```bash
npm install --save-dev nyc
```

## 配置 NYC 工具:

在 package.json 中添加 NYC 的配置:

```json
"nyc": {
  "reporter": [
    "text",
    "lcov"
  ],
  "all": true,
  "check-coverage": true,
  "lines": 80,
  "statements": 80,
  "functions": 80,
  "branches": 80
}
```

- reporter 指定输出文本摘要和 lcov 格式的覆盖率报告。
- all 表示覆盖所有文件,包括未测试的文件。
- check-coverage 启用覆盖率检查。
- lines、statements、functions、branches 分别设置各个指标的覆盖率阈值为 80%。

## 修改测试脚本:

在 package.json 的 scripts 中添加使用 NYC 运行测试的脚本:

```json
复制
"scripts": {
  "test": "nyc mocha 'test/**/*.test.js'"
}
```

## 生成覆盖率报告:

运行 npm test 后,NYC 会在项目根目录生成 coverage 目录,里面包含文本摘要和 lcov 格式的覆盖率报告。 您也可以使用第三方工具(如 Coveralls 或 CodeCov)来托管和展示覆盖率报告。

## 持续集成中使用覆盖率:

在前面配置的 GitHub Actions 工作流中,添加生成并检查覆盖率的步骤:

```yaml
- name: Run unit tests with coverage
  run: npm run test
- uses: coverallsapp/github-action@master
  with:
    github-token: ${{ secrets.GITHUB_TOKEN }}
    path-to-lcov: ./coverage/lcov.info
```

这样,每次提交都会自动生成覆盖率报告,并将其上传到 Coveralls 等第三方服务。
通过以上步骤,您就可以在 Node.js 项目中设置测试覆盖率阈值并生成覆盖率报告了。

一些补充说明:

- 设置合理的覆盖率阈值非常重要,过高可能会造成开发效率下降,过低则无法保证代码质量。通常 80% 是一个较为合理的目标。
- 除了查看总体覆盖率外,也要关注各个指标(行、语句、函数、分支)的覆盖情况,确保代码质量。
- 除了 NYC,还可以使用 Istanbul 等其他覆盖率工具,选择合适的工具并熟练使用是关键。
- 持续优化测试覆盖率是一个长期的过程,需要与团队持续沟通并不断完善。
